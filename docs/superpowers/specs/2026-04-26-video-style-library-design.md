# 视频风格库设计

## 状态

设计草案，等待确认。本文定义 `video-master` 的可扩展视频风格库：用户可以选择完全原创，也可以选择一个已经维护好的导演风格模板。

## 产品目标

把一次性的参考片分析，沉淀成可复用的导演方法库。风格模板不是复制某支参考片的 prompt，而是一套可迁移的创作系统，包含节奏、视觉语言、分镜策略、声音策略和提示词规则。

## 用户入口

### 完全原创

用户提供想法、素材、时长、比例、语言和目标模型。`video-master` 从零设计风格、结构、剧本、分镜图和提示词。

### 使用正式模板

用户从风格库里选择一个已经确认过的模板。模板会影响制作锁定、节奏图、剧本结构、分镜策略、图片提示词、视频提示词和声音规则。

### 基于参考片创建新模板

用户提供参考视频或参考图片。skill 先分析素材并生成草稿模板，再接收用户补充的设计思路、剪辑技巧和创作意图。只有用户确认后，模板才进入正式库。

## 模板状态

### 草稿模板

由 AI 分析生成，允许不完美。草稿模板可以包含不确定观察、候选规则和待确认问题。除非用户明确选择，否则草稿模板不能作为正式项目的默认依据。

### 正式模板

经过用户或维护者确认。正式模板会出现在正常项目入口里，可以稳定用于生成新项目。

## 模板套用强度

正式模板支持三档强度：

### 轻度参考

只借用氛围、色调、光影和整体气质。新项目保留自己的结构和镜头策略。

### 中度套用

默认模式。借用风格、节奏结构、镜头语言、声音策略和提示词规则，但重新设计具体故事、人物、产品和画面。

### 高度风格锁定

强继承模板的叙事结构、节奏压力点、镜头语法和提示词结构。仍然不能复制参考片的具体画面、台词、人物、品牌、水印或创作者身份。

## 模板包结构

每个模板独立成一个文件夹：

```text
skills/video-master/style_templates/<template_id>/
  template.md
  template.json
  rhythm_rules.json
  prompt_rules.md
  reference_notes.md
  examples/
```

### `template.md`

给人看的导演说明书。解释这个模板的情绪核心、适用场景、节奏、视觉语言、声音设计、分镜策略和安全边界。

### `template.json`

给 skill 稳定读取的结构化规则。包含模板 id、名称、状态、版本、标签、适用题材、不适合题材、三档强度行为、必要门槛和相关文件引用。

### `rhythm_rules.json`

可复用的节奏语法。记录阶段结构、推荐镜头数量、快切组、呼吸镜头、高潮位置、镜头能量和转场规则。

### `prompt_rules.md`

图片提示词和视频提示词规则。定义必须继承的风格语言、禁止出现的 prompt 写法、声音处理、字幕策略和模型适配说明。

### `reference_notes.md`

参考分析记录。说明从参考片里学到了什么、哪些内容不能复制，以及人工补充的创作思路。

### `examples/`

可选示例，比如示范分镜、示范提示词或总览图。示例只用于理解模板，不能作为复制源。

## 正式模板必填信息

每个正式模板必须包含：

- 模板 id 和展示名称。
- 状态：`draft` 或 `official`。
- 版本和更新时间。
- 用于检索的标签。
- 适用题材和不适合题材。
- 支持的视频模式、画面比例和时长范围。
- 轻度、中度、高度三档套用规则。
- 视觉风格规则。
- 节奏和剪辑规则。
- 镜头和构图规则。
- 声音和口播规则。
- 分镜图生成提示词规则。
- 视频生成提示词规则。
- 安全边界和不可复制项。

## 第一枚模板候选

模板 id：`cinematic-flow-racing`

展示名称：`意识流赛车压迫感短片`

核心方法：用高反差黑白画面、极近机械细节、主观驾驶压力、快切压力段和后段超现实心流转场，表现人物从恐惧、过载到进入绝对专注的心理变化。

适合题材：

- 赛车手独白。
- 极限运动员。
- 拳击手或格斗运动员。
- 关键手术前的外科医生。
- 演出前的钢琴家。
- 高压交易员或创业者。
- 任何带有高压职业和心流转变的主题。

不适合题材：

- 明亮美妆广告。
- 轻松电商种草。
- 亲子儿童内容。
- 喜剧短片。
- 直接信息讲解类视频。

## 参考片入库流程

基于参考片创建模板时：

1. 标记素材角色：`reference_style`、`reference_subject` 或 `do-not-copy`。
2. 提取视频信息、关键帧、总览图、色彩、亮度、饱和度、对比度、运动强度和候选剪辑点。
3. 可选分析音频能量、字幕区域、画面文字和节奏变化。
4. 生成草稿风格档案，并区分“有证据的观察”和“创意解读”。
5. 生成草稿节奏规则、分镜策略、声音规则和提示词规则。
6. 询问用户是否补充设计思路、剪辑技巧、情绪目标、必须保留的原则，以及 AI 可能漏掉的细节。
7. 合并 AI 分析和人工补充，形成草稿模板包。
8. 用户确认后，草稿模板才升级为正式模板。

## 与当前流程的整合

### Step 1：输入阶段

增加一个早期选择：

- 完全原创。
- 使用正式风格模板。
- 基于参考片创建新模板。

如果用户选择模板，把 `template_id` 和 `template_strength` 写入 `brief/spec_lock.md`。

### Step 2：制作锁定

制作锁定必须说明模板会如何影响：

- 视频模式。
- 时长和画面比例。
- 故事节奏。
- 视觉风格。
- 声音和字幕策略。
- 分镜图覆盖范围。
- 提示词语言。

### Step 3：节奏图

如果选择模板，模板的节奏规则先作为第一版基础。skill 可以根据新主题和时长调整，但明显偏离模板时必须说明原因。

### Step 4：剧本和声音

模板的声音策略决定口播是画外音、人物现场说话、纯视觉叙事还是混合形式。外部画外音必须留在音频和 SRT 文件里，不能直接写进最终视频生成提示词。

### Step 5：分镜和提示词

模板的图片和视频提示词规则会注入每个镜头。最终提示词仍然必须围绕用户的新主题重写，不能复制参考片画面或原文。

## 校验规则

校验器需要检查：

- 选择的模板 id 是否存在。
- 是否在没有明确授权时使用了草稿模板。
- 是否缺少模板套用强度。
- 正式模板是否缺少标签、适用题材或不适合题材。
- 最终提示词是否复制了禁止复制的参考内容。
- 最终提示词是否违反模板级字幕、口播、音乐或 SFX 规则。
- 节奏图是否声称使用模板，但没有继承必要节奏规则，也没有解释偏离原因。

## 测试策略

增加聚焦测试：

- 能正确加载模板包。
- 能拒绝信息不完整的正式模板。
- 只有明确授权时才允许使用草稿模板。
- 能把轻度、中度、高度三档强度写入制作锁定。
- 能把模板规则注入图片和视频提示词。
- 能验证外部画外音不会进入最终视频提示词。
- 能验证第一枚 `cinematic-flow-racing` 模板具备必要元数据和规则。

## 第一版实现范围

第一版先做文件型模板库，不做 UI：

1. 给 skill 增加 `style_templates/`。
2. 增加模板读取工具。
3. 增加第一枚正式模板：`cinematic-flow-racing`。
4. 更新 `SKILL.md` 的入口和流程规则。
5. 更新校验器，让它理解模板选择。
6. 增加测试。

UI、模板搜索、模板评分、云端读视频服务、多模板混合，放到后续扩展。

## 暂不处理的开放问题

- 未来是否允许一个项目混合两个模板。
- 用户个人模板是否可以存放在 skill 文件夹外。
- 模板创建时是否强制生成一张模板预览总览图。
- 模型专用提示词变体应该放在每个模板里，还是继续放在全局平台配置里。
